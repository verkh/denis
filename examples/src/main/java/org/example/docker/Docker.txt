Docker - легковесное средство виртуализации

Варианты виртуализации:
    - Виртуальные машины
    - контейнеры (Docker, podman, etc.)

######### Виртуальные машины:
Используются очень активно и распространены широко.

Преимущества:
    - Полная изолированность (с точки зрения секурити)

Недостатки:
    - Медленная загрузка
    - Потребляет много ресурсов
    - Нужен гостевой клиент для взаимодействия хоста и ВМ
    - Тяжеловесное
    - Требует установки
    - Требует большие образы

Основные характеристики:
    - Полноценная операционная система 
    - Изолирована от основной операционной системы
    - Отсюда нужен коммуникатор между основной ОС и виртуальной

######## Контейнеры, в частности Docker

Преимущества:
    - леговесность, которая включает в себя, что образ весит очень мало  (https://hub.docker.com/)
    - потербляет мало ресурсов: мало ЦПУ и мало памяти за счет того, что запускается только самое нужное
    - не требует установки гостевого клиента

Недостаток:
    - низкая изолированность

Основная идея какая:
    Изначально докер запускался только на Linux системах
    Докер - это процесс, которрые построен на CGroups и napespaces
    И по сути докер - это часть операционный системы.


Термины:
Хост/Хост Машнина/Хост ОС   - та машина или ОС, которая собственно и запускает контейнер
Containers                  - это сами котнейнеры, твои процессы внутри контейнеров. Изолированные процессы.
Images                      - это образ контейнера, который будет использоваться для создания
Volumes                     - позволяет замапить папку в


Команды:
docker -v                   - показать версию докера
docker ps                   - показать все контейнеры, которые запущены сейчас,
docker ps -a                - показать совсем всен контейнеры
docker pull <image name>    - скачать имадж докера docker pull alpine
docker image ls             - показать все существующие образы на текущей машине
docker image rm <image id>  - удалить образ

docker run <image> <commad> - запустить контейнер и зайти в него используя баш
    docker run -it alpine bash
        run     - запустить
        -it     - интерактивный режим
        alpine  - образ, который будет использован
        bash    - команда

docker exec <container id> <commad> - подключиться к существующему контейнеру
    docker exec -it <container_id> sh

docker rm <container id>    - удалить контейнер по имени

docker volume ls            - просмотреть все вольюмы
docker network ls           - посмотреть все нетворки настроенные
